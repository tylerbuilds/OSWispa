03 / Processing and privacy

Choose how speech is processed

“Local” and “remote” mean different things here.

There is no account and no product analytics. The selected processing mode still determines where microphone audio goes.

Read the complete privacy notice →
LocalProcessed on this computer

After the first model download, Whisper.cpp transcribes on the computer running MorpheOS Voice. Each recording uses an owner-only temporary WAV that is deleted on success, cancellation and handled failure paths.

RemoteSent to the selected provider

If you explicitly configure remote mode, the audio file and request fields are sent to your chosen OpenAI-compatible endpoint. That provider's security and retention terms apply.

Completed transcripts can be stored in bounded local history. Audio being captured at the instant of a process or operating-system crash is not recoverable.

05 / Questions

Useful answers

What this product is—and is not.

Is MorpheOS Voice fully local?

Local processing is the default and stays on the computer after a model has been downloaded. Optional remote processing sends audio to the endpoint you choose, so the product is not described as universally local.

Does it control my computer?

No. Write mode produces and inserts text. There is no action or autonomous-agent mode in the current product.

Does it learn from corrections?

No automatic learning occurs. You can maintain an explicit local personal vocabulary for names and phrases.

Why do downloads still say oswispa?

The first MorpheOS Voice transition release retains the established executable, packages and data locations so existing users keep settings, shortcuts, models, history and rollback options.

Is this affiliated with OpenAI?

No. MorpheOS Voice is an independent open-source project. It uses Whisper.cpp locally and can optionally call an OpenAI-compatible endpoint chosen by the user.
