Speak. Release. Keep writing.

OSWispa is a small dictation layer between your microphone and your cursor. Hold one hotkey, say a short thought, and release. Whisper transcribes locally by default, then copies and inserts the text into the app you are already using.

A deliberately small tool

Built for the sentence, not the meeting.

Use OSWispa for replies, prompts, notes and code comments. It is designed for short dictation bursts, not hour-long recordings or cloud document memory.

Private by architecture

Local by default means local by default.

The desktop app has no account or product analytics. In local mode, microphone audio and transcripts stay on the computer running OSWispa.

Read the privacy notice
  1. Temporary audio

    Each recording uses an owner-only temporary WAV that is deleted after transcription, including error paths.

  2. Local state

    Configuration and transcript history stay in OS-specific application directories rather than one universal path.

  3. Remote is explicit

    Audio leaves the device only when you select remote mode and configure an endpoint. That endpoint’s privacy policy then applies.

Release proof, with boundaries

Checksummed packages, tested as packages.

Every release asset is covered by SHA256SUMS. Clean hosted VMs install and launch both macOS DMGs and the Windows ZIP before publication.

What that does not prove

Hosted VMs do not prove physical microphones, operating-system permission prompts, global keypresses, focus-sensitive insertion or GPU acceleration. Those remain hardware checks.

Setup help

The edges worth knowing before download.

What happens on first launch?

OSWispa runs a short device probe, selects a responsive Whisper model and downloads it. The application needs network access for that first model download; local dictation can work offline afterwards.

Why does Linux copy text but not insert it?

Wayland insertion depends on the installed ydotoold user service. Check it without root:

systemctl --user status ydotoold
journalctl --user -u ydotoold -n 50

The transcript remains on the clipboard if insertion fails.

Why do macOS and Windows show security warnings?

The current alpha packages are not signed. macOS may require Control-click → Open; Windows may require SmartScreen’s More info → Run anyway. Only download from the official GitHub release.

Where are settings and models stored?

Locations differ by operating system. Linux has tray-based settings; macOS and Windows currently use their OS-specific configuration files. The user guide lists platform behaviour without assuming a Linux home-directory path.

What should I try for [BLANK_AUDIO] on Linux?

Confirm the active PipeWire/PulseAudio source, then choose the working source in Linux Settings if needed:

pactl get-default-source
pactl list short sources
