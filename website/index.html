Changelog

What changed in v0.4.2 on July 18, 2026.

The audited desktop release now has checksummed Linux, macOS and Windows downloads, plus package-install proof on clean hosted VMs.

  • Configuration, history, API-key fallback files, temporary recordings, IPC and remote responses now follow stricter privacy and failure-handling rules.
  • Model downloads reject server errors, partial files, path traversal and invalid Whisper model headers before installation.
  • Windows now records through WASAPI, listens for the global Ctrl+Windows hotkey, verifies clipboard delivery and inserts text into the focused app.
  • Both macOS DMGs and the Windows ZIP are installed and executed on clean hosted VMs before publication.
  • Linux packages are built on Ubuntu 22.04 for a GLIBC 2.35 compatibility floor, with Debian and RPM requirements checked automatically.
  • Every public artefact is covered by the release's SHA256SUMS file and a post-publication reinstall gate.

Install

Get running in minutes.

Download packages for x86-64 Linux, Apple Silicon and Intel macOS, or x86-64 Windows.

Install (.deb)

Ubuntu/Debian users: download the .deb from GitHub Releases and install with apt.

v0.4.2: built on Ubuntu 22.04 for a GLIBC 2.35 compatibility floor.

curl -LO https://github.com/tylerbuilds/OSWispa/releases/latest/download/oswispa_amd64.deb
sudo apt install ./oswispa_amd64.deb
oswispa
Prefer source builds? Run ./install.sh from the repo.

Install (macOS)

Mac users get a drag-to-Applications app bundle. On first launch, OSWispa runs a short device test and downloads the model that best fits the machine.

1. Download the matching macOS .dmg
2. Drag OSWispa.app to Applications
3. Open OSWispa from Applications
4. If macOS blocks the first launch:
   Control-click the app and choose Open
Apple Silicon can use Metal. Intel Macs stay on the lighter CPU path. The app launches in Terminal and will ask for Microphone + Accessibility permissions.

Install (Windows)

Download the x86-64 ZIP, extract it into a permanent folder, and run oswispa.exe.

1. Download oswispa-windows-x86_64.zip
2. Extract the complete ZIP
3. Run oswispa.exe
4. Hold Ctrl+Windows to dictate
Download the Windows ZIP. The alpha is unsigned and has no tray UI yet.

Auto paste tips

Wayland uses ydotoold for direct text insertion into the focused app. Clipboard copy is still kept as a fallback.

systemctl --user status ydotoold
journalctl --user -u ydotoold -n 50
If text does not appear, start the daemon or use the copied transcript from the clipboard.

FAQ

Common questions answered.

Troubleshooting tips and setup guidance for OSWispa.

OSWispa doesn't start on login

The most common cause is a missing or unreachable model file. Check your config at ~/.config/oswispa/config.json and verify the model_path exists. Models should be stored locally at ~/.local/share/oswispa/models/.

For reliable autostart, use a systemd user service instead of the desktop file: 

# Create the service
cat > ~/.config/systemd/user/oswispa.service <<EOF
[Unit]
Description=OSWispa Voice-to-Text
After=graphical-session.target

[Service]
Type=simple
ExecStart=/usr/local/bin/oswispa
Restart=on-failure

[Install]
WantedBy=default.target
EOF

# Enable and start
systemctl --user daemon-reload
systemctl --user enable --now oswispa

Model not found error

OSWispa exits immediately if the configured model file is missing. If your models are on a network drive or external mount, they may not be available at login time.

Fix: Copy models to local storage:

mkdir -p ~/.local/share/oswispa/models
cp /path/to/your/model.bin ~/.local/share/oswispa/models/

Then update ~/.config/oswispa/config.json to point to the local path.

App closes when terminal closes

If you're running oswispa from a terminal, closing that terminal kills the app. Use the systemd service approach above, or launch with nohup: 

nohup oswispa & disown

The systemd service is recommended for production use.

OSWispa inserts [BLANK_AUDIO] or reports no speech

On Linux, OSWispa follows the default PipeWire/PulseAudio input unless you choose a source in Settings → General → Linux microphone source. 

pactl get-default-source
pactl list short sources
pactl set-default-source SOURCE_NAME

Use the working source name in Settings if you want to override the default only for OSWispa.

Text doesn't appear automatically

On Wayland, automatic insertion depends on ydotoold: 

sudo systemctl enable ydotool
sudo systemctl start ydotool

If paste still fails, text is always copied to the clipboard. Use Ctrl + V manually.

How do I change the hotkey?

Open Settings from the tray icon, then configure modifiers and an optional trigger key. You can still edit ~/.config/oswispa/config.json directly: 

"hotkey": {
  "ctrl": true,
  "alt": false,
  "shift": true,
  "super_key": false,
  "trigger_key": "space"
}

Changes apply immediately.

How do I check if OSWispa is running?

Check the systemd service status or look for the process:

systemctl --user status oswispa
# or
pgrep -f oswispa

The system tray icon also indicates active status.

Can I use a VPS instead of local models?

Yes. In Settings, switch Backend mode to Remote VPS, set your endpoint, and optionally store an API key securely. HTTPS is required by default.

OSWispa remains local-first and can fall back to local models when available.
