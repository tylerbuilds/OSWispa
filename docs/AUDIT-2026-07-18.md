# OSWispa audit — 18 July 2026

## Scope and baseline

This audit covered the Rust application, Linux and macOS recording paths, clipboard and hotkey integration, local persistence, optional remote transcription, model management, source installer, Linux packages, macOS bundle workflow, website, CI, and release automation.

The baseline release was [v0.4.1](https://github.com/tylerbuilds/OSWispa/releases/tag/v0.4.1). Findings fixed after that tag are in `master` and remain unreleased until a new version is tagged. The assumed deployment is a public alpha used on personal desktops, with microphone audio and transcripts treated as privacy-sensitive data.

## Dependency review

Compatible lockfile updates removed advisories affecting `anyhow` 1.0.101 and `memmap2` 0.9.10, and removed the future-incompatible `wl-clipboard-rs` 0.8 line. CI runs `cargo audit` on every change.

Two RustSec advisory IDs remain explicitly accepted:

- [`RUSTSEC-2026-0194`](https://rustsec.org/advisories/RUSTSEC-2026-0194)
- [`RUSTSEC-2026-0195`](https://rustsec.org/advisories/RUSTSEC-2026-0195)

Both affect `quick-xml` versions below 0.41. The 0.37 line is a Windows-notification transitive dependency that is not present in OSWispa's Linux build. The 0.39 line is used by `wayland-scanner` at build time to parse trusted, bundled protocol XML; it does not parse microphone, transcript, clipboard, model, or remote-response data. The latest compatible upstream `wayland-scanner` still uses a vulnerable `quick-xml`, so these IDs are ignored in CI pending an upstream upgrade. Reassess this exception whenever Wayland dependencies change.

`cargo audit` also reports unmaintained or unsound transitive warnings in the legacy GTK/KSNI/Reqwest dependency lines. The relevant code paths do not use the reported GLib iterator API, while the `atty` warning is in a build-time generator. These remain migration debt rather than confirmed exploitable application paths.

## Supply-chain and release controls

- GitHub Actions are pinned to full commit SHAs.
- Dependabot is configured for Cargo and GitHub Actions updates.
- CI now gates formatting, clippy with warnings denied, headless tests, default builds, installer fixtures, desktop metadata, `.deb`, `.rpm`, and RustSec audit status.
- Release tags must match `Cargo.toml` and point to a commit contained in `master`.
- Linux release builds use Ubuntu 22.04 to reduce the GLIBC compatibility floor.
- A single publish job verifies the complete seven-asset set and publishes SHA-256 checksums.
- RPM and Debian runtime dependencies are asserted before release artefacts are uploaded.

## Verification evidence

- 35 Rust tests pass in both headless and default-feature configurations; two hardware tests remain ignored.
- Linux default and no-default-feature builds pass.
- Clippy passes with `-D warnings` for the default Linux target.
- Linux, macOS, and Windows GitHub Actions passed for PRs #17, #18, and #19.
- Shell installer fixtures, shell syntax, desktop-file validation, Rust formatting, and diff checks pass.
- [Release rehearsal 29652797663](https://github.com/tylerbuilds/OSWispa/actions/runs/29652797663) passed from merged `master`: it built and uploaded the Linux tarball, Debian package, RPM package, and Arm64/Intel macOS ZIP and DMG artefacts. Publication was correctly skipped because the run was not triggered by a tag.
- GitHub vulnerability alerts, automated security-update pull requests, private vulnerability reporting, secret scanning, push protection, and a pinned Rust CodeQL workflow were enabled after the code audit.

## Accepted limitations and follow-up

- Linux `/dev/input` hotkeys require the broad `input` group permission. A portal/compositor-native hotkey path remains future work.
- macOS packages are not signed or notarised.
- Windows compiles and tests but has no implemented audio or hotkey runtime and remains experimental.
- Microphone, clipboard, feedback-sound, GPU-accelerated transcription, installer privilege changes, and desktop focus behaviour require hardware/manual release testing.
- VAD configuration exists but no recorder currently emits `VadSilenceDetected`; it is not advertised as a working feature.
- The Settings model-download row does not refresh itself after a background download; reopening Settings shows the validated model.
