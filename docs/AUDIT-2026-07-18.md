# OSWispa audit — 18 July 2026

## Scope and baseline

This audit covered the Rust application, Linux, macOS, and Windows recording paths, clipboard and hotkey integration, local persistence, optional remote transcription, model management, source installer, desktop packages, website, CI, and release automation.

The current release baseline is [v0.4.2](https://github.com/tylerbuilds/OSWispa/releases/tag/v0.4.2). The original audit fixes through PR #29 shipped in that release; later productisation and platform hardening in PRs #33–#41 remain Unreleased on `master` until another version is tagged. The assumed deployment is a public alpha used on personal desktops, with microphone audio and transcripts treated as privacy-sensitive data.

## Dependency review

Compatible lockfile updates removed advisories affecting `anyhow` 1.0.101 and `memmap2` 0.9.10, removed the future-incompatible `wl-clipboard-rs` 0.8 line, and updated `rust-openssl` from 0.10.75 to 0.10.80. The OpenSSL update closes five high-, two medium-, and one low-severity memory-safety alerts indexed by GitHub in the remote HTTPS dependency path. CI runs `cargo audit` on every change.

Two RustSec advisory IDs remain explicitly accepted:

- [`RUSTSEC-2026-0194`](https://rustsec.org/advisories/RUSTSEC-2026-0194)
- [`RUSTSEC-2026-0195`](https://rustsec.org/advisories/RUSTSEC-2026-0195)

Both affect `quick-xml` versions below 0.41. The 0.37 line is a Windows-notification transitive dependency that is not present in OSWispa's Linux build. The 0.39 line is used by `wayland-scanner` at build time to parse trusted, bundled protocol XML; it does not parse microphone, transcript, clipboard, model, or remote-response data. The latest compatible upstream `wayland-scanner` still uses a vulnerable `quick-xml`, so these IDs are ignored in CI pending an upstream upgrade. Reassess this exception whenever Wayland dependencies change.

`cargo audit` also reports unmaintained or unsound transitive warnings in the legacy GTK/KSNI/Reqwest dependency lines. The relevant code paths do not use the reported GLib iterator API, while the `atty` warning is in `dbus-codegen`, a build-time dependency of the tray library. GitHub tracks these as [`GHSA-wrw7-89jp-8q8g`](https://github.com/advisories/GHSA-wrw7-89jp-8q8g) and [`GHSA-g98v-hv3f-hcfr`](https://github.com/advisories/GHSA-g98v-hv3f-hcfr); they are accepted migration debt rather than confirmed exploitable application paths. The GLib fix requires a GTK dependency-line migration, and no patched `atty` release exists.

## Supply-chain and release controls

- GitHub Actions are pinned to full commit SHAs.
- Dependabot is configured for Cargo and GitHub Actions updates.
- `master` requires pull requests, up-to-date Linux/macOS/Windows/CodeQL checks, resolved conversations, and linear history; force pushes and deletion are blocked.
- CI now gates formatting, clippy with warnings denied, headless tests, default builds, installer fixtures, desktop metadata, `.deb`, `.rpm`, and RustSec audit status.
- Release tags must match `Cargo.toml` and point to a commit contained in `master`.
- Linux release builds use Ubuntu 22.04 to reduce the GLIBC compatibility floor.
- A single publish job verifies the complete seven-asset set and publishes SHA-256 checksums.
- RPM and Debian runtime dependencies are asserted before release artefacts are uploaded.

## Accepted limitations and follow-up

- Linux `/dev/input` hotkeys require the broad `input` group permission. A portal/compositor-native hotkey path remains future work.
- The v0.4.2 macOS packages are not signed or notarised. Unreleased capture code adds per-session lifecycle recovery, full CPAL sample-format conversion, and streaming anti-alias downsampling ([PR #41](https://github.com/tylerbuilds/OSWispa/pull/41)); a signed build, microphone permission, physical input, rapid global-hotkey use, and focused-app insertion still require native proof.
- Windows v0.4.2 includes WASAPI recording, a global Ctrl+Windows push-to-talk hotkey, verified clipboard delivery, text insertion, and installed-package hosted-VM proof ([PR #29](https://github.com/tylerbuilds/OSWispa/pull/29)). The ZIP remains unsigned and lacks an installer or tray UI; a physical microphone, global-hotkey, focus-sensitive insertion, and permission pass remains required.
- Microphone, feedback sound, GPU-accelerated transcription, installer privilege changes, and physical focus behaviour remain hardware/manual release checks; VM clipboard and runtime-backend smoke checks do not prove those paths.
- VAD configuration exists but no recorder currently emits `VadSilenceDetected`; it is not advertised as a working feature.
- The Settings model-download row does not refresh itself after a background download; reopening Settings shows the validated model.
